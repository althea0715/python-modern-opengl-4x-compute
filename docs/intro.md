저는 데이터 사이언티스트로서 데이터를 분석하고 어떤 모델을 세울때 데이터를 통해 최적화를 하는 업무를 주로 합니다.
제가 하는 일은 결국 시간이라는 요소가 배제되게 됩니다. 
경력이 깊어지는 만큼 다른 도메인 지식을 쌓고 싶은데, 그러다가 수치 미분을 활용하는 방식의 해석 기법들을 사용할 수 있는 계기가 되었습니다.
제 성향상 어떤 툴에 익숙할 시간에 프로그래밍 언어로 직접 필요한 부분만 구현하는 것을 선호합니다.
많은 수치 미분을 활용한 기법들은 메쉬를 활용하게 되고 시간에 따라 변화하는 것을 봐야하므로 결국 동영상을 만드는 상황이 되어버립니다.
이때 선택할 수 있는 방법은 OpenGL, DirectX, Vulkan, WebGPU(WebGL)이 있습니다. 
저는 윈도우에서만 개발하기 때문에 DirectX를 선택하는 것이 맞긴 하지만 연구자로써 언제나 쓸데없이 리눅스도 고려해야한다고 생각하였습니다.
그래서 모든 곳에서 추천하는 OpenGL을 LearnOpenGL을 통해 먼저 공부했습니다. 제가 공부해왔던 방향과는 매우 다른 방향으로 매우 재미있는 시간이었습니다.
이제 그래픽이 뭔지 알았다고 해야겠죠. 저는 최적화를 하기 때문에 항상 연산 속도를 따집니다. 그러면 이제 Vulkan으로 가야겠죠.
아, Vulkan은 그냥 쉽게 공부할 수가 없습니다. 그래픽이 뭔지 아는 것과 다른 너무 많은 고려사항을 알아야합니다.
저와 같은 사람은 고속 연산을 위해 GPU를 쓰는 방법을 이미 알고 있고 꼭 Vulkan의 Compute Shader를 써야할 이유가 없죠.
당연히 C++을 항상 공부하고 있기 때문에 최신의 C++ 버젼을 활용하고 CUDA도 활용가능하지만 누가 요즘 C++로 처음부터 끝까지 개발합니까?
당연히 병목부분만 어떻게든 numpy 쓰다가 cython 쓰다가 어쩔수 없이 C++을 활용하죠. CUDA도 cupy, torch로 충분히 대응 가능하구요. jax는 윈도우에선 아쉽습니다.
다시 본론으로 돌아와서 OpenGL을 해야할 것 같습니다. OpenGL을 정말 다 알았다싶으면 천천히 Vulkan으로 넘어가면 될거 같습니다. 
근데 OpenGL을 C++로 glad, glfw 등을 빌드해가며 해야하나요? 너무 피곤합니다. UI 개발 해본다면 알다싶이 뭐 기능하나 만들면 확인해보고 해야하는 절차를 거쳐야합니다.
OpenGL로 게임을 개발하는 업을 갖추신 분들은 그렇지 않겠죠. 저도 제 엔지니어링에 OpenGL을 항상 사용해왔다면 그랬을 수 있겠습니다만, 저도 계속 빌드를 해보며 봐야하는 사람입니다.
C++보단 Python이 결과물 보는데 빠르잖아요. Python으로 OpenGL을 하는 방법은 옛날부터 구축되어 있었습니다. 여튼 OpenGL은 끝난 프로젝트니깐요.
제가 원하는 2D 혹은 3D 메쉬를  시각화하는 겁니다. 기왕이면 그래픽카드에서 병럴 처리해서 보여주면 좋겠죠.
OpenGL4.3 부터는 제가 원하는게 되는 것 같습니다. 그렇다면 그걸 공부해야해요.
앗, Modern OpenGL은 3.3입니다! OpenGL4.3은 무엇인가요? 나름의 여러 정보들을 찾아봤지만 누군가 구현하기 위해 노력한 흔적은 찾기 힘들었습니다.
